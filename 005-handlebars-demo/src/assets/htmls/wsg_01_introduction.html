Introduction

The goal of the Web Style Guide is to inform the developers how all the graphical user interface elements and mechanisms should behave and look like. These guidelines either apply to entire webpages used in our projects, or to individual user interface components alike.

It is beneficial to have this style guide in order to create a cohesive experience amongst different pages. Also it helps to ensure that future development or third-party production will follow The Web Style Guide guidelines

What is a Web Design Guide?

These pages are a style guide that is formed out of a collection of pre-designed elements, graphics and rules that designers and developers should follow during the creation of web projects. This style guide ensures that the web projects will look and behave in a consistent fashion. This will improve the user experience with the targeted audience.

Why Is It Important?

It is extremely important when multiple resources are working on a big website or web app to ensure that they don't interpret too much and don't alter or adjust styles based on personal preference. In development, having defined elements of the website makes it easy for developers to reuse these elements. Moreover, it can make it easier because they will get what elements they have to code and will see exactly how they need to look from the start.

In order to make developers lives easier, it is the designer's duty to include all possible interactions such as hover, click, visit and other states for buttons, titles, links, etc.
