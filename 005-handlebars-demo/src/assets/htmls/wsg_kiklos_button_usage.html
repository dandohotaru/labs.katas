Button and Link Usage

Problem a variety of buttons are used in KIKLOS. So when and how do we use them consistently.
Solution first come up with KIKLOS-styled buttons. Next, define a common set of rules on when and how to use them correctly.


Buttons vs textual links

Use buttons on the most important call to actions, and links for less important ones.

Primary call to action buttons

Each primary call to action button should have a meaningful icon, label and colour. The KIKLOS red is used to suggest people to interact with the button. The white button means cancel this action or go back.

Contextual call to action buttons

To avoid overkill-to-action, the red KIKLOS colour is absent when we use contextual CTA's. Besides a meaningful label name, the buttons are acompanied with meaningful icons, rendering the site less boring or administrative.

The User Menu

KIKLOS makes use of a horizontal User Menu to harbour lesser actions. It displays the following:

  1. The KIKLOS logo in the top left corner of the page. This logo is omniprensent throughout the entire KIKLOS experience and clicking on it takes the user back to the homepage. No HTML text label is needed for this.
  2. A number of minor menu items, including a dropdown menu to reach scarcer menu items.
  3. More immediate follow-up actions are grouped in a red badge, grabbing the user's attention and involvement.
