Common UI components

This (unfinished) list describes the global user interface elements we use to build our webpages.

  1. The KIKLOS Header
    1. Brand exposure KIKLOS
    2. Brand baseline KIKLOS
    3. Primary Navigation Home - Funding opportunities - Experts - ...
    4. Secondary Navigation This is only displayed on subpages as a vertical menu
    5. Recurrent Menu Know as the top menu: contains language choice, link to sitemap ...
    6. Breadcrumbs This provides the context of the KIKLOS ecosphere
  2. Homepage - General
    1. Homepage Top Just below the Primary Navigation, the Mission Statement is placed.
    2. Homepage Bottom Information Carousel. This UI component reuses the screen real estate.
  3. Subpages
    1. Information Architecture The total amount of the IA is spread out over x-amount of subpages. (Funding Opportunities, How to participate,...)
    2. Mission Statement Each subpage has a Mission Statement container at the top. There, within a limited amount of characters the purpose of this page is explained.
  4. Headings h1, h2, h3, h4, h5, h6 and variations of typographic headings
  5. Button Usage - Colours
    1. Blue used to navigate (Next/Prev) or primary call to action buttons (Save, Register as an Expert,...)
    2. White/Grey used for contextual decission-making (pagination, filtering, sorting, datepicker...)
    3. Green used to finalize a decission making process (wizard, registration,...)
    4. Yellow/Red used in very important decission-making processes (delete, discard,...)
  6. Button Usage - Positioning
    1. Blue when used as primary call to action buttons, it is centered at the bottom
    2. Blue when used navigate, it is aligned on the right at the bottom
  7. Button Usage - Sizes
    1. Blue to be defined...
    2. Blue to be defined...
  8. Button Usage - Functional
    1. Active to be defined...
    2. Disabled to be defined...
    3. loading... to be defined...
  9. Search Mechanism - Global allows the user to search across the entire site
  10. Search Mechanism - Contextual or local allows the user to search a specific area
  11. Login Mechanism positioned at the top right, adjacent with the primary navigation and below the Portal Header.
  12. Footer positioned at the bottom of each webpage. Contains general recurrent information (copyright, author, contact information,...)
  13. Area Effects - Jumbotron Used as a big banner and attention grabber (as seen in the landing page of URF and the Web Design Guide)
  14. Area Effects - Page Header This has been replaced by the Mission Statement.
  15. Area Effects - Wells Used to put emphasis on information by placing information inside a grey or blue area.
  16. Lists Unordered, ordered, definition, bulleted, numbered... Any collection of elements presented in a list-type format
  17. Tables Used to display tabular data.
  18. Tables - Interactive Tabular data can be paginated, filtered, sorted and searched (jQuery DataTables Plugin)
  19. Cards these information cards provide a creative and responsive solution to display tabular data.
  20. Blocks a collection of headings and/or images and/or information excerpts.
  21. Pagination a solution to reuse the screen real estate and allow the user to navigate through a lot of information.
  22. Tooltips Meant to provide contextual help.
  23. Forms - General Used to retrieve information from the user.
  24. Forms - Validation Used to make sure that the required information asked is valid.
  25. Forms - Login Techniques
  26. Image types to be defined...
    1. Graphics photos and graphics
    2. Logos the E.C. logo
    3. Avatars to be defined...
    4. Thumbnails to be defined...
    5. Controllers special types of images worthy of their own category (magnifying glasses, social icons, arrows, hamburgers, spinners, favicons,...)
  27. Messages and notifications Alerts, success, errors, warnings, errors, validation, in-progress, popups, tooltips, 4o4s, etc. This is a challenging category as these messages often require user action to expose, but it’s essential to get messaging right
  28. Wizards and Setup Assistants to be defined...
  29. Sitemap A mandatory webpage that provides a summary and entry points to the total amount of information of the Participant Portal. Also needed for web accessibility reasons.
  30. Interactive Components to be defined...(Accordions, tabs, carousels, and other modules with moving parts)
  31. Media to be defined...(Video players, audio players and other rich media elements)
  32. 3rd Party to be defined...(Widgets, iframes, stock tickers, social buttons, anything that isn’t hosted on your domain)
  33. Advertising to be defined...(A special kind of 3rd party category which includes all ad formats and dimensions)
  34. Animation and interactive design to be defined...(transitions, attention-grabbing...)
